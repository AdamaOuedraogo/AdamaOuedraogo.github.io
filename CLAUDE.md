# Personal Website Vision

Before writing any code, I want you to understand the purpose of this website.

This is NOT a portfolio.

This is NOT a resume.

This is NOT a personal homepage.

It is the digital headquarters of my professional identity.

Everything should be designed around this idea.

---

# Background

I recently finished almost six years as a Senior QA Automation Engineer at Aircall.

Instead of simply looking for another QA position, I decided to use this transition to completely reinvent my professional positioning.

I am documenting this journey publicly.

My objective is to become recognized as an:

**AI-Powered Quality Engineering Consultant**

specialized in:

* Agentic Quality Engineering
* AI for QA
* MCP
* AI-assisted testing
* Playwright
* AI workflows
* Software Quality Strategy

The website should communicate this transformation.

---

# Philosophy

Most portfolios describe the past.

This website should describe the future.

It should answer one question:

"Why should someone follow Adama?"

Not:

"What jobs has Adama done?"

---

# Positioning

The website should communicate:

I help engineering teams prepare for the future of Software Quality.

Not simply by writing automated tests.

But by helping them understand and adopt:

* AI
* MCP
* Agentic Workflows
* Modern Quality Engineering

---

# The feeling

Visitors should feel that they are entering an innovation laboratory.

Not reading a CV.

The website should inspire curiosity.

It should make people think:

"This is someone exploring where Quality Engineering is going."

---

# Design Principles

Minimal.

Modern.

Technical.

Elegant.

Calm.

Lots of whitespace.

No flashy animations.

No unnecessary graphics.

Content first.

Thinking first.

Learning first.

# Content Strategy

Everything should reinforce my positioning.

Every experiment can become:

* a LinkedIn post
* a YouTube video
* an article
* a GitHub repository
* a consulting offer
* a workshop
* a training lesson

The website should naturally connect all these assets.

# Relationship with Agentic Quality Lab

My Agentic Quality Lab project is my operating system.

This website is its public face.

The website should eventually consume content generated by the lab.

Examples:

Latest experiments

Latest articles

Latest learning

Latest GitHub projects

Future videos

No duplication of work.

The lab produces.

The website showcases.

Preferred stack:

* Next.js
* TypeScript
* Tailwind CSS
* MDX
* Vercel

No CMS.

Markdown-first.

Easy to maintain.

Easy to extend.

Always ask:
